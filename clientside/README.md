# TelegramFS Client Side

Клиентская часть распределённой файловой системы TelegramFS, реализующая FUSE-интеграцию.

## Назначение

Client Side позволяет смонтировать удалённое хранилище TelegramFS как локальную файловую систему через FUSE (Filesystem in Userspace).

## Основные компоненты

### FUSE реализация
- **TelegramFS** — основной класс, расширяющий FuseStubFS
  - `getattr()` — получение атрибутов файла
  - `readdir()` — чтение содержимого директории
  - `open()` / `release()` — открытие и закрытие файлов
  - `read()` / `write()` — чтение и запись файлов
  - `mkdir()` / `create()` / `mknod()` — создание файлов и директорий
  - `unlink()` / `rmdir()` — удаление файлов и директорий
  - `truncate()` — обрезка файлов
  - `rename()` — переименование файлов

### Модели данных
- **NodeInfo** — информация об узле файловой системы
- **NodeAttributes** — атрибуты узла
- **Privileges** — права доступа
- **ChangeFile** — запрос на изменение файла
- **RenameRequest** — запрос на переименование
- **TruncateRequest** — запрос на обрезку
- **ResponseMessage** — ответ от сервера

### Сервисы
- **FileSystemService** — интерфейс файловой системы
- **TelegramFileSystemService** — реализация клиента для взаимодействия с backend

### Bash-скрипты
- **tgfs.sh** — основной скрипт запуска
- **run.sh** — скрипт запуска Java приложения
- **lazy_tgfs.sh** — ленивый режим запуска

## Технологии

- Java 21
- Spring Boot 3.2.5
- Spring WebFlux
- jnr-fuse 0.5.7
- Lombok
- Gradle (Kotlin DSL)

## Сборка

```bash
./gradlew build
```

Создание JAR со всеми зависимостями:
```bash
./gradlew uberJar
```

## Запуск

```bash
cd bash
./tgfs.sh
```

## Поддерживаемые операции

- Чтение и запись файлов
- Создание и удаление файлов
- Создание и удаление директорий
- Переименование файлов и директорий
- Просмотр содержимого директорий
- Изменение размера файлов (truncate)

## Требования

- Установленный FUSE в системе
- Запущенный backend сервис
